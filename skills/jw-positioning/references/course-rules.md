# 定位基座｜公开运行规则

- Capability: `C00`
- Public release: `v1.0.0-rc1`
- Provenance: 由私有课程真源编译并完成独立评测；公开版省略来源编号、路径、哈希和内部审批记录。

只把 `origin=COURSE_NEW` 的判断当作课程规则。把 `COURSE_INTERPRETATION` 保留为已批准解释，把 `AI_ENGINEERING` 只用于结构、路由、版本、审批和失败控制。不得用验证案例改写规则。

## 决策规则

| ID | Origin | 编译判断 |
|---|---|---|
| C00-D01 | COURSE_NEW | 把定位首先视为消费者为什么选你的商业选择，不以流量问题替代。 |
| C00-D02 | COURSE_NEW | 判断线上客户真正关心、具有价值且值得进入的需求方向。 |
| C00-D03 | COURSE_NEW | 识别竞争者与替代选择已占据的心智，判断可选的独特购买理由/心智空位前提。 |
| C00-D04 | COURSE_NEW | 将创始人身份、经历、表达/表现、交付和资源条件纳入方向判断。 |
| C00-D05 | COURSE_NEW | 在需求、竞争心智与创始人自身条件的交集中形成候选并保留取舍理由。 |

## 不变量

| ID | Origin | 必须保持 |
|---|---|---|
| C00-I01 | COURSE_NEW | 定位必须同时考虑线上真实需求、竞争心智与创始人自身条件。 |
| C00-I02 | COURSE_NEW | 定位首先是商业方向与打法的取舍，不是流量热度、一句人设或什么火就做什么。 |
| C00-I03 | COURSE_NEW | 批准方向不得越过创始人真实交付、表达、身份和资源条件。 |
| C00-I04 | AI_ENGINEERING | C00 只向 C10 交付定位与产品边界，不决定最终四锁人群、四圈、购买理由、理念或制作。 |
| C00-I05 | AI_ENGINEERING | 定位基线必须携带取舍理由、未确定项、字段级来源、批准状态、版本与哈希。 |

## 边界

- 不以流量热度、热门赛道或单条爆款代替商业定位
- 不把一句人设、职业或宽泛人群标签当完整定位
- 不越过创始人真实交付、表达和资源条件
- 不替 C10 决定最终产品可成交人群与买单前阶段
- 不设计 C20 四圈、C30 购买理由、C40 理念或 C50 制作方案
