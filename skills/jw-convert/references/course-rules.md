# 线索成客｜公开运行规则

- Capability: `C30`
- Public release: `v1.0.0-rc1`
- Provenance: 由私有课程真源编译并完成独立评测；公开版省略来源编号、路径、哈希和内部审批记录。

只把 `origin=COURSE_NEW` 的判断当作课程规则。把 `COURSE_INTERPRETATION` 保留为已批准解释，把 `AI_ENGINEERING` 只用于结构、路由、版本、审批和失败控制。不得用验证案例改写规则。

## 决策规则

| ID | Origin | 编译判断 |
|---|---|---|
| C30-D01 | COURSE_NEW | 将目标从关注、喜欢或认可校正为产生买单欲望与购买动作 |
| C30-D02 | COURSE_NEW | 识别目标人无法直接判断真实专业度以及过深表达可能劝退的差距 |
| C30-D03 | COURSE_NEW | 只用真实身份、关系、结果与交付事实组织目标人可理解的专业和靠谱感觉 |
| C30-D04 | COURSE_NEW | 从目标用户可感知的世界形成为什么选择你的即时购买理由 |
| C30-D05 | COURSE_NEW | 区分信任型与成交型内容，并用相似客户的真实决策过程形成进一步笃定 |
| C30-D06 | COURSE_NEW | 仅在产品确有购买机会时为已信任人群设置通知型角色，不以破圈流量判断其价值 |
| C30-D07 | COURSE_NEW | 将理念、信任、成交、通知作为多条内容的旅程角色组合，不创建固定条数或单条全包规则 |

## 不变量

| ID | Origin | 必须保持 |
|---|---|---|
| C30-I01 | COURSE_NEW | 目标人和买单前阶段必须继承 C10 批准版本，C30 不得重定义。 |
| C30-I02 | COURSE_NEW | 专业事实由真实交付承接，专业感只能组织可感知表达，不能制造能力。 |
| C30-I03 | COURSE_NEW | 信任不等于成交；成交还必须形成可感知购买理由与笃定。 |
| C30-I04 | COURSE_NEW | 购买理由必须从目标用户可理解、可感知的世界表达，并由真实事实承接。 |
| C30-I05 | COURSE_NEW | 四类内容是多内容旅程角色，不允许单条全包或伪造固定条数。 |
| C30-I06 | AI_ENGINEERING | C30 只持有理念型角色指针，理念本体与理念表达由 C40 持有。 |
| C30-I07 | COURSE_NEW | 通知型视频只告知已信任人群真实存在的购买机会，不以破圈流量判断其价值。 |
| C30-I08 | AI_ENGINEERING | 通知型视频不得与 内部批准记录（公开版省略） 已排除的付费产品介绍合并或互相补写。 |

## 边界

- 不修改 C10 已批准人群或买单前阶段
- 不修改 C20 四圈语义或用愿意看代替愿意买
- 不创造真实专业、身份、关系、结果或交付事实
- 不创建 C40 理念本体、理念表达或长期一致性规则
- 不创建 C50 场景、设备、机位、妆造或灯光方案
- 不把通知型视频混同已排除的付费产品介绍
