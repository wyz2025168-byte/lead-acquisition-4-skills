# 线索锁客｜公开运行规则

- Capability: `C10`
- Public release: `v1.0.0-rc1`
- Provenance: 由私有课程真源编译并完成独立评测；公开版省略来源编号、路径、哈希和内部审批记录。

只把 `origin=COURSE_NEW` 的判断当作课程规则。把 `COURSE_INTERPRETATION` 保留为已批准解释，把 `AI_ENGINEERING` 只用于结构、路由、版本、审批和失败控制。不得用验证案例改写规则。

## 决策规则

| ID | Origin | 编译判断 |
|---|---|---|
| C10-D01 | COURSE_NEW | 由已批准产品反推可成交人群候选 |
| C10-D02 | COURSE_NEW | 检查正价成交事实，不伪造稳定天数、单量或金额阈值 |
| C10-D03 | COURSE_NEW | 锁定认为问题值得当前价格与额外成本的产品可成交人群 |
| C10-D04 | COURSE_NEW | 在可成交候选中锁定创始人真正熟悉其心理、纠结、疑问和表达的人 |
| C10-D05 | COURSE_NEW | 锁定目标人当前的买单前具体阶段，不把买单后交付阶段当获客默认内容 |
| C10-D06 | COURSE_NEW | 锁定买单前关心的价值、方向、生活或行业话题，排除默认展开的买单后解决方法 |
| C10-D07 | COURSE_NEW | 将四锁合并成唯一可向下传递的人群、阶段与话题边界 |

## 不变量

| ID | Origin | 必须保持 |
|---|---|---|
| C10-I01 | COURSE_NEW | 人群必须在 C00 已批准产品与业务边界内。 |
| C10-I02 | COURSE_NEW | 不得把所有有需求者、泛职业标签或所有观看者当产品可成交人群。 |
| C10-I03 | COURSE_NEW | 正价交易只记录事实和未验证状态，不创建跨行业统一数字阈值。 |
| C10-I04 | COURSE_NEW | 四锁必须合并为同一批人的交集，不得拼接不同人群。 |
| C10-I05 | COURSE_NEW | 买单前阶段与话题不得默认滑入买单后交付方法。 |
| C10-I06 | AI_ENGINEERING | C10 不输出四圈、购买理由、拍摄方案或理念。 |
| C10-I07 | AI_ENGINEERING | 无可追溯正价交易事实时只能产出 PROJECT_HYPOTHESIS 候选，不得批准或发送 I-02。 |
| C10-I08 | AI_ENGINEERING | 批准版锁客基线只能有一个主目标人群；多个候选同时可行时保留差异并交用户取舍。 |

## 边界

- 不修改 C00 已批准定位、产品或业务方向
- 不设计 C20 开头、画面、内容或观点圈
- 不创建 C30 购买理由、成交内容或消费旅程
- 不创建 C50 拍摄、场景或设备方案
- 不用播放量、泛留资或引流品代替正价品成交事实
