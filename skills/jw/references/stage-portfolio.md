# 内容配比系统

调用：`conversion-journey`、`belief-mindshare`、`orchestration-control`；用 `audience-continuity` 检查同一人，用 `production-presentation` 核验库存可实现性。

## 目标

让内容库存覆盖客户从识别到行动的关键障碍，并使拍摄前计划与拍摄后实际资产一致。比例是当前项目的资源配置假设，不是课程通用公式。

## 拍摄前规划

1. 按目标人和客户关卡列出本周期必须解决的内容任务。
2. 盘点每个任务已有证据、可拍人物、制作成本、承接价值和风险。
3. 生成至少两个组合方案：稳态组合、重点突破组合；保留维持现状作对照。
4. 按业务优先级、当前瓶颈、证据成熟度、素材产能和渠道需求配置计划权重。
5. 检查是否过量生产理念、泛科普、通知或单一人物内容，导致关键购买障碍无人处理。
6. 用 `add-content` 登记每条计划的客户阶段、旅程任务、目标 claim、格式、计划权重和依赖。

## 拍摄后核对

1. 用 `record-asset` 将实际素材关联到原内容计划，不因拍得好看而重新定义任务。
2. 运行 `reconcile-portfolio`，比较计划权重、实际素材数量、可发布状态和旅程覆盖。
3. 把差异分成：可以重剪、需要补拍、需要补证、上游已失效、暂时不应发布。
4. 评估未来30天是否足够支持稳定发布，以及90天是否形成内容结构失衡或人物/场景依赖。

## 交付物

- 计划内容组合及每个比例的项目依据；
- 实际素材库存和可用状态；
- 客户关卡覆盖图；
- 计划/实际差异；
- 重剪清单、补拍清单、补证清单和禁止发布清单；
- 下一周期只调整哪一个组合假设。

## 判断边界

单条爆款不能自动提高同类内容长期比例；播放量不能替代目标人质量和关卡信号；素材数量不能替代可发布性；上游 claim 失效后，依赖它的内容库存必须 stale。
