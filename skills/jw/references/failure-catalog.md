# 失败目录

| 失败 | 识别 | 返回 |
| --- | --- | --- |
| 结果替代 | 成功事件等于咨询、预约、到店、检查或方案沟通 | B0 |
| 线下即线上 | 有需求或线下可成交直接推出线上可转化 | B4 |
| 内部视角人设 | 职务和内部职责直接推出客户购买 | B6/B7 |
| 泛研究越权 | E3/E4 单独批准本地排序 | B5 |
| 数据工程转嫁 | 索取全量资料却不能说明改变哪个决定 | 当前 IRQ 目标节点 |
| 内容抢跑 | B5 未批准就批准 C1/C2/P1 | B5 |
| 模糊审批扩张 | “继续看看”被解释成方向或发布批准 | 重新确认审批 |
| 领先指标冒充交易 | 播放、咨询、到店改善就宣称业务成功 | B0/O1 |

每个失败输出固定包含错误码、记录 ID、违反关系、原因和最小修复；结构 PASS 不能覆盖业务 FAIL。
