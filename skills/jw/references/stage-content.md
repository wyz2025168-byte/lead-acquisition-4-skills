# 内容系统

## 目标

让已确定目标人在特定购买阶段完成一个可观察的心理或行动跃迁，并把策略转成可直接拍摄和承接的内容，而不是只生成选题名称。

## 输入门槛

需要有效的产品边界、目标人、角色、购买阶段、替代选择和合规边界。没有真实客户语言时，内容只能标为验证表达；人物身份、职务、权限和可兑现触点必须有来源。

## 判断程序

1. 指定唯一目标人、当前状态、下一状态和当前障碍。
2. 分开六个问题：现在、品类、人物、兑现信任、行动、方案/付款；确定本条内容只处理哪一项。
3. 建立竞争解释：客户不行动可能是无需求、时点不对、不信品类、不信人物、风险过高、价格/家庭决策或承接失败。不要默认是“认知不足”。
4. 为目标跃迁写内容机制、所需证据、可观察信号、否证信号和失败去向。
5. 设计开头、画面、正文和观点，使同一目标人持续留下；逐段检查是否换成同行或泛人群。
6. 将内部专业翻译成客户可感知事实，保留范围、失败和不确定性；不得把结果证明写成保证。
7. 比较至少两个表达方向及维持现状，按相关性、可信度、行动阻力、可拍性、合规和长期心智选择。
8. 生成完整成品，并模拟它对拍摄成本、内容组合、CTA 承接和90天内容心智的影响。

## 交付物

- 客户关卡：`from_state / to_state / barrier / mechanism / required_evidence / observable_signal / disconfirming_signal / failure_route`；
- 选题矩阵：目标人、阶段、六问位置、内容角色、证据、目标信号、错误人群风险；
- A/B 开头及唯一变量；
- 完整口播脚本，不只给大纲；
- 逐镜头画面意图和人物动作；
- CTA、咨询承接话术和不适配说明；
- 验证卡：最小样本、周期、阈值、误导指标、停止与扩展条件；
- 对拍摄、配比和运营的明确约束。

## 内容角色

- 识别：让正确的人认出阶段，不代表信任；
- 解释：建立对问题/品类的正确理解，不代表选择你；
- 信任：用真实行为和交付信号证明可能兑现，不代表适合；
- 决策：处理相似客户买前比较、风险和取舍，不代表付款；
- 行动：给已就绪者明确、低歧义入口；
- 理念：累积长期认同，不能独占内容组合。

## 停止条件

上游定位失效、目标人和阶段不唯一、脚本依赖抽象词、人物事实被猜测、关卡没有机制或证据、素材未授权、CTA 超出承接能力时停止进入拍摄。
