# 结果合同

结果合同是一等实体，不是一句 KPI。先明确业务真正优化的价值交换，再允许上游判断进入批准状态。

## 必填含义

- `core_offer`：卖什么、价格依据、容量和不做什么。
- `success_event`：真实正价或业务等价价值交换；必须给证据与归因窗。
- `leading_events`：咨询、预约、到店、检查、方案沟通等过程事件，以及它们为何不等于成功。
- `excluded_success_events`：明确禁止冒充成功的事件。
- `economics`：最低可行利润、容量和可重复性依据。

“正价”由项目定义，可是付款、合同生效或同等真实价值交换；系统不统一定价。非一次性现金业务也必须定义真实交换，不得退化为点击、问价或咨询。

未批准 B0 时可以探索 B1–B4，但不得批准 B5、C1、C2、P1 或 O1。新结果口径推翻旧口径时，依赖旧口径的内容、实验和指标失效，原始来源与无关事实保留。
