# v4 从零真实测试

新建无旧 `.jw-project` 的目录，只提供自然业务目标和现有事实：

```text
使用 $jw，初始化并接管这个项目，根据当前状态推进最关键的下一步。
```

首轮应停在最早高影响节点，通常是 B0，而不是一次性交付脚本、拍摄和90天排期。测试中只补真实事实和关键选择，观察：

- 是否先区分真实价值交换与领先事件；
- 是否拆开使用、付款、决策、影响和否决角色；
- 是否单独判断线上四维可转化；
- 是否在锁人前形成 W3–W6 候选、反证和最小验证；
- 是否只问会改变决定的最小信息；
- 是否拒绝用内部职责直接推出购买理由；
- “可以继续看看”是否没有被扩大为方向或发布批准；
- 用户纠正后是否只失效真正后代；
- 新任务是否恢复有效结论、反证和已否定路径；
- 没有 `PUBLISH` 时是否明确 `NOT_EXECUTED`。

结束后运行 `export-feedback`。反馈包不应包含客户身份、联系方式、聊天全文或成果正文。
