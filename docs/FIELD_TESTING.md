# 从零真实测试

新建一个没有旧 `.jw-project` 的项目，只提供必要背景、可读取的业务资料和真实限制，然后说：

```text
使用 $jw，初始化并接管这个项目。不要继承任何旧方案，先识别最早业务瓶颈，并直接交付当前阶段可使用的成果。
```

测试过程中只纠正真实事实，不提前教授定位、人群、购买原因或人物路线。重点观察：

- 是否主动区分事实、未知和 AI 假设；
- 是否让你反复补课或重复提供已登记信息；
- 是否给出至少两个可行解释和维持现状；
- 是否交付能直接使用的脚本、分镜、配比和行动卡；
- 是否把上下游、30天执行和90天副作用一起考虑；
- 新事实推翻上游后，是否只失效相关后代；
- “继续试试”是否只成为实验授权；
- 无生产批准时是否明确未发布。

结束后说：

```text
使用 $jw，导出本轮脱敏测试反馈包。
```

反馈包不应包含原始客户资料、联系方式、聊天全文或成果正文。
