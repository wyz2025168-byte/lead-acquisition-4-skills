# 从 v3 升级到 v4

1. 不覆盖原项目；运行 `migrate PROJECT --to 4`。
2. 运行时先把原 `.jw-project` 原子移动为带时间戳的只读备份，再建立 v4。
3. 来源和已确认事实可迁移；旧业务决定统一待复核。
4. 缺少用户原话、解释范围和排除项的旧审批变为 `NEEDS_RECONFIRMATION`。
5. 旧项目将咨询、预约、到店、检查或方案沟通视为成功时，只迁为领先事件；B0 重新定义。
6. 旧人物路线若只依赖职务或内部职责，必须重新通过客户侧测试。
7. 重复迁移不会重复建备份或改写状态；`migration-report.json` 保存映射、降级和回滚说明。
8. 如需恢复迁移前状态，运行 `python3 skills/jw/scripts/jw_project.py rollback-migration <项目目录>`。旧项目恢复为活动目录，当前 v4 状态整体移入隔离快照，不删除任何一边。

运行时不会自动删除备份、发布内容或联系客户。
