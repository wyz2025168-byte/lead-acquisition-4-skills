# Phase 3 结果

- 改动文件：七个 `method-*.md`、五个 `stage-*.md`、`method-router.md`、`method-registry.json`、研究与行业适配参考。
- 新增行为：五阶段只负责用户导航；七方法显式声明读取节点、写入节点、证据要求和停止条件。
- 先失败的测试：旧方法可直接从内部职责跳到人物结论，也没有 B4 线上可转化和 W3–W6 前置检查。

- 仍然失败：无确定性失败。
- 业务规则如何进入运行时：定位阶段强制 B0–B4；内容阶段必须读取已批准 B5–B7；生产、配比和运营必须沿节点依赖追溯。
- 对 v3 兼容影响：保留五阶段和七方法，但旧阶段完成状态不能伪造新节点批准。
- 风险与下一步：行业适配器提供决策结构，不替代本地一手事实或实时规则核验。
