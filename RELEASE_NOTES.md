# v4.0.0-rc.1 Candidate Notes

这是保留 v3 项目控制层、重建业务推理层和评测层的语义候选，不覆盖 `v3.0.0` 历史。

## 主要变化

- 新增 OutcomeContract 和 B0–O1 原子业务节点。
- 新增结构化 alternatives、InformationRequest 和 Approval v4。
- 默认 `validate` 同时执行结构检查和机器业务不变量。
- 新增 `preflight`、节点级失效、业务型 HANDOFF、v3→v4 幂等迁移和非破坏式回滚。
- W3–W6 成为 B5 前的强制候选预检；人物路线必须通过客户侧去头衔测试。
- 新增线上四维可转化、证据适用范围、研究冲突和最小信息门禁。
- 口腔适配器增加交易角色、项目差异、支付路径和线上可影响模型。
- 重建自然项目、反例、迁移、污染扫描和脱敏高客单回放。

## 候选状态

确定性业务测试、脱敏回放、包级门禁、隐私扫描和干净安装均已通过。第二轮21次独立外部模型运行全部成功，19/21单次运行、6/7场景稳定通过，硬失败0；当前可进入受控 Field Validation。

该版本不保证流量、咨询或成交；没有 `PUBLISH` 不会执行外部动作。未经用户另行授权，本轮不 commit、不 push、不发布 GitHub Release。
